# Zombie Courier (좀비 택배) — 게임 기획서 v1.0

> **작성일:** 2026-02-21
> **장르:** 횡스크롤 러너 (Casual / Action)
> **플랫폼:** 모바일 & 데스크탑 웹 브라우저 (Vercel 배포)

> **타겟 유저:** 캐주얼 모바일 게이머 (15–35세)

## 1. 게임 개요

### 1.1 한 줄 소개
좀비 아포칼립스 세계관에서 오토바이 택배 기사가 되어 생존자들에게 물품을 배송하는 횡스크롤 러너 게임.

### 1.2 엘리베이터 피치
"세상이 멸망해도 택배는 온다." 플레이어는 좀비 떼가 들끓는 폐허 위를 오토바이로 질주하며, 생존자 거점에 물품을 배송한다. 장애물을 넘고, 좀비를 처치하고, 한정된 연료를 관리하며 더 먼 거리를 달리는 것이 목표다. 점수와 배송 기록을 친구와 공유하며 경쟁한다.

### 1.3 핵심 경험 (Target Aesthetics — MDA 프레임워크)

| MDA Aesthetic | 구현 |
|---|---|
| **Challenge** (도전) | 점진적 난이도 상승, 순간 판단력 요구 |
| **Sensation** (감각) | 속도감, 파티클 이펙트, 화면 흔들림, 좀비 처치 피드백 |
| **Fantasy** (판타지) | 좀비 아포칼립스 세계에서 영웅적 택배 기사 |
| **Discovery** (발견) | 새 구간, 히든 아이템, 잠금 해제 콘텐츠 |
| **Fellowship** (사회적 교류) | 리더보드, 점수 공유, 친구 기록 도전 |

### 1.4 플레이어 유형 타겟 (Bartle + Quantic Foundry)

| 유형 | 우선순위 | 대응 시스템 |
|---|---|---|
| **Achiever (성취자)** | 1순위 | 업적, 수집, 진행 시스템, 리더보드 |
| **Explorer (탐험가)** | 2순위 | 히든 아이템, 신규 구간, 잠금 해제 |
| **Killer (경쟁자)** | 3순위 | 글로벌 랭킹, 친구 기록 도전 |
| **Socializer (사교가)** | 4순위 | 점수 공유, 소셜 경쟁 |

## 2. 코어 루프

```
[출발] → [주행 + 장애물 회피 + 좀비 처치] → [아이템 수집] → [거점 도착 = 배송 완료]
   ↑                                                                    ↓
   └── [보상 획득 → 업그레이드 → 다음 배송] ←─────────────────────────────┘
```

### 2.1 세션 흐름 (1회 플레이: 60~180초)

1. **출발** — 배송 미션 확인, 출발 (0.5초)
2. **주행** — 자동 전진, 플레이어는 점프/슬라이드/공격 조작 (30~150초)
3. **배송 완료 또는 게임 오버** — 거점 도착 시 보상 / 체력 0 시 게임 오버
4. **결과 화면** — 점수, 거리, 획득 아이템, 공유 버튼
5. **메타 루프** — 코인으로 업그레이드, 다음 미션 선택

### 2.2 세션 길이 설계 근거
- 모바일 UX: 2~3분 단위 세션 = 출퇴근/대기 시간 적합
- 플로우 이론: 짧은 세션 → 빠른 피드백 사이클 → 몰입 유지
- 자이가르닉 효과: "한 판 더"를 유도하는 최적 길이

## 3. 게임 메카닉

### 3.1 이동 & 조작

| 입력 | 데스크탑 | 모바일 |
|---|---|---|
| **점프** | ↑ / W / Space | 화면 상단 탭 |
| **슬라이드** | ↓ / S | 화면 하단 스와이프 다운 |
| **공격** | J / 클릭 | 화면 오른쪽 탭 |
| **부스트** | K / Shift | 부스트 버튼 |

**설계 원칙:**
- 모바일 우선: 엄지 영역(thumb zone) 내 모든 핵심 조작
- 터치 타겟: 최소 48×48px (Apple HIG 44pt + 여유)
- 원핸드 가능: 세로 화면에서도 플레이 가능하도록 설계
- 입력 즉각성: 입력 → 반응 ≤ 50ms (Game Feel)

### 3.2 주행 시스템

- **자동 전진**: 오토바이는 자동으로 우측 이동 (기본 속도: 300px/s)
- **속도 변화**: 레벨 진행에 따라 기본 속도 증가 (DDA 적용)
- **지형**: 평지, 오르막, 내리막, 파괴된 도로, 점프대
- **연료 게이지**: 시간 경과로 감소 → 연료 아이템으로 보충 → 긴장감 유지

### 3.3 장애물 & 적

| 가로등/표지판 | 점프 또는 슬라이드 | 2구간~ |
| 폭발물 통 | 좌우 회피 (다중 레인) | 3구간~ |
| 떨어지는 잔해 | 타이밍 점프 | 4구간~ |

#### 좀비 유형
| 좀비 | 행동 | HP | 등장 시점 |
|---|---|---|---|
| **일반 좀비** | 느린 이동, 직선 돌진 | 1 | 1구간~ |
| **러너 좀비** | 빠른 이동, 추격 | 2 | 2구간~ |

| 무기 | 공격력 | 범위 | 특징 |

## 5. 난이도 설계

### 5.1 난이도 곡선 (Flow Channel 설계)

```
난이도
  │          ╱ 불안 영역 (Frustration)
  │        ╱
  │      ╱    ★ Flow Channel
  │    ╱        (도전 ≈ 능력)
  │  ╱
  │╱        지루함 영역 (Boredom)
  └──────────────────── 플레이 시간/레벨
```

### 5.2 구간별 난이도 파라미터

| 파라미터 | 구간 1 | 구간 2 | 구간 3 | 구간 4 | 구간 5 |
|---|---|---|---|---|---|
| 기본 속도 | 300 | 350 | 400 | 430 | 460 |
| 좀비 밀도 (/100m) | 2 | 4 | 6 | 8 | 10 |
| 장애물 밀도 (/100m) | 3 | 5 | 7 | 9 | 12 |
| 장애물 복잡도 | 단일 | 2연속 | 3연속 | 혼합 | 고밀도 혼합 |
| 안전 구간 비율 | 40% | 30% | 20% | 15% | 10% |

### 5.3 동적 난이도 조절 (DDA)

**추적 지표:**
- 최근 3회 플레이의 평균 거리
- 회피 성공률
- 게임 오버 빈도

**조절 규칙:**
```
IF 연속 3회 게임오버 && 평균거리 < 해당구간 기대거리의 50%:
  → 좀비 밀도 -20%, 안전구간 +10%, 아이템 드롭률 +30%

IF 연속 3회 클리어 && 평균거리 > 해당구간 기대거리의 150%:
  → 좀비 밀도 +10%, 속도 +5%, 특수 좀비 비율 증가

IF 회피성공률 > 90%:
  → 복합 장애물 빈도 증가
```

**핵심:** DDA는 은밀하게 적용 — 플레이어가 인지하면 부정적 반응 (연구 근거).

### 5.4 인지 부하 관리

- **새 메카닉 1개씩 도입**: 구간 1에서 점프만 → 구간 2에서 슬라이드 추가 → 구간 3에서 원거리 무기
- **점진적 공개 (Progressive Disclosure)**: UI 요소 단계적 노출
- **시각적 경고**: 새 장애물/적 첫 등장 시 슬로모션 + 가이드 표시

## 6. 아트 스타일

### 6.1 방향: 모던 픽셀 아트 (16-bit Inspired)

**선택 근거:**
- 소규모 팀에서 높은 생산성 (벡터 대비 에셋 제작 3~5배 빠름)
- 좀비 아포칼립스 장르와 뛰어난 조화 (Dead Cells, Katana ZERO 입증)
- 웹 게임에 최적화된 작은 파일 사이즈 (~300KB 이하 전체 스프라이트)
- 독특한 시각적 정체성 — 캐주얼 유저에게도 차별화된 인상

### 6.2 아트 스펙

| 요소 | 사양 |
|---|---|
| **해상도 스타일** | 16-bit 모던 픽셀 (32×32 ~ 64×64 캐릭터) |
| **캔버스 해상도** | 480×270 (16:9) → 디스플레이 스케일업 |
| **컬러 팔레트** | 제한 팔레트 (32~48색), 좀비/폐허 테마 |
| **애니메이션** | 프레임 기반 스프라이트 (주행 8프레임, 공격 6프레임) |
| **배경** | 3~4레이어 패럴렉스 스크롤링 |
| **이펙트** | 파티클 시스템 (먼지, 피, 폭발, 불꽃) |

### 6.3 컬러 팔레트 방향

```
메인 톤: 어두운 회색/갈색 (폐허)
악센트:  오렌지/빨강 (불꽃, 위험)
         네온 그린 (좀비 독소, UI)
         파랑 (아이템, 안전)
UI:      네온 그린 + 화이트 (좀비 아포칼립스 시그니처)
```

### 6.4 패럴렉스 배경 레이어 구성

| 레이어 | 속도 비율 | 내용 |
|---|---|---|
| Layer 4 (최후방) | ×0.1 | 하늘 + 먼 빌딩 실루엣 |
| Layer 3 | ×0.3 | 중경 건물 + 연기 |
| Layer 2 | ×0.6 | 근경 건물 + 디테일 |
| Layer 1 (최전방) | ×1.0 | 도로 + 장애물 + 캐릭터 |
| Layer 0 (오버레이) | ×1.2 | 전경 파티클 (먼지, 안개) |

## 7. 사운드 디자인

### 7.1 음악

| 상황 | 스타일 | 템포 |
|---|---|---|
| 메인 메뉴 | 어두운 앰비언트 + 저음 신스 | 80 BPM |

| 구간 5 (보스) | 에픽 오케스트라 + 디스토션 | 160 BPM |
| 게임 오버 | 피아노 단독 + 역재생 앰비언트 | 60 BPM |

### 7.2 효과음 (Game Feel / Juice)

| 이벤트 | 효과음 | 시각 효과 동반 |
|---|---|---|
| 점프 | 오토바이 엔진 rev | 먼지 파티클 |
| 착지 | 타이어 충격음 | 스쿼시&스트레치 |
| 좀비 타격 | 둔탁한 타격음 + 좀비 비명 | 히트스톱(30ms) + 화면 흔들림 |
| 좀비 폭발 (뚱뚱이) | 폭발음 + 스플래시 | 파티클 + 화면 흔들림(강) |
| 코인 수집 | 밝은 징글 (피치 에스컬레이트) | 숫자 팝업 (+10) |
| 부스트 발동 | 제트 엔진 + 와프 | 스피드 라인 + 잔상 |
| 니어미스 | 긴장감 사운드 + 슬로 | 줌인 + 모노톤 플래시 |
| 배송 완료 | 팡파레 징글 | 별 파티클 + UI 애니메이션 |
| 게임 오버 | 크래시 사운드 + 좀비 소리 | 화면 균열 이펙트 |

### 8.2 UX 휴리스틱 적용 (Nielsen + PLAY)

| 휴리스틱 | 적용 |
|---|---|
| **시스템 상태 가시성** | HP, 점수, 연료, 콤보 항상 표시. 위험 시 화면 가장자리 붉은 비네팅 |
| **현실-시스템 일치** | 빨강=위험/체력, 초록=좀비/독, 파랑=아이템, 금=코인 |
| **사용자 제어와 자유** | 일시정지 항상 가능, 즉시 재시작, 메뉴 복귀 |
| **일관성** | 같은 색=같은 의미, 모든 화면에서 동일 조작 |
| **오류 예방** | 유료 구매 시 확인 대화상자 |
| **인식 > 회상** | 무기 잔탄/연료 HUD 상시 표시, 버튼에 아이콘+텍스트 |
| **유연성** | 데스크탑 키보드/모바일 터치 자동 감지 |
| **미학적 미니멀리즘** | HUD 최소화, 핵심 정보만 표시 |
| **오류 복구** | 게임오버 → 1탭으로 즉시 재시작 |
| **도움말** | 첫 플레이 시 인게임 튜토리얼, 신규 요소 등장 시 가이드 |

### 8.3 온보딩 설계 (FTUE — First Time User Experience)

**원칙: 3탭 이내에 코어 루프 경험**

| 단계 | 내용 | 시간 |
|---|---|---|
| 1 | "탭하여 시작" → 즉시 주행 시작 | 0초 |
| 2 | "탭하여 점프!" (첫 장애물) | 3초 |
| 3 | "탭하여 공격!" (첫 좀비) | 8초 |
| 4 | 자유 플레이 (30초간 낮은 난이도) | 15초 |
| 5 | 배송 완료 → 보상 → "축하! 첫 배송 완료!" | 45초 |

- 튜토리얼 중 게임오버 불가능 (체력 자동 회복)
- 부여된 진행 효과: 첫 배송 완료 시 코인 500 + 오토바이 스킨 즉시 지급

### 8.4 소셜 공유 기능

| 기능 | 구현 |
|---|---|
| **점수 공유** | Web Share API (`navigator.share()`) |
| **결과 이미지** | Canvas 스크린샷 → 커스텀 결과 카드 |
| **URL 공유** | Open Graph 메타 태그로 링크 프리뷰 최적화 |
| **챌린지 링크** | `zombiecourier.app/challenge?score=23450&user=player1` |
| **리더보드** | 주간/월간/전체 랭킹 (친구 필터 가능) |
